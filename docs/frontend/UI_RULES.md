# UI_RULES.md — 前端视觉与样式规范

## 1. 视觉方向
整体视觉采用“学院数据中枢”风格：清晰、可信、理性，不使用花哨拟物，不走默认紫色系。

## 2. 品牌与色彩
建议 CSS 变量：
```css
:root {
  --bg-base: #f4f7fb;
  --bg-panel: #ffffff;
  --bg-emphasis: #e8f1ff;
  --text-primary: #10243e;
  --text-secondary: #4b5f78;
  --text-muted: #7b8ca4;
  --line-soft: #d9e3f0;
  --brand-main: #0f5bd8;
  --brand-accent: #0ea5a0;
  --ok: #1f9d55;
  --warn: #e5a100;
  --danger: #cf3f3f;
}
```

配色约束：
- 主操作使用 `--brand-main`。
- 状态色仅用于状态表达，不用于大面积装饰。
- 背景用浅灰蓝渐层，不用纯白大平面。

## 3. 字体规范
- 中文主字体：`"Source Han Sans SC", "Noto Sans SC", "PingFang SC", sans-serif`
- 英文与数字辅助字体：`"IBM Plex Sans", "Segoe UI", sans-serif`
- 代码与 ID：`"JetBrains Mono", "Cascadia Mono", monospace`

说明：
- 禁止默认 `Inter/Roboto/Arial` 作为唯一字体方案。
- 标题与正文需明确字号层级，不做全站同字号。

## 4. 排版与间距
- 8pt 栅格：`4/8/12/16/24/32`
- 页面最大宽度建议 `1200~1360px`
- 主内容区建议左右留白 `24~32px`
- 卡片圆角建议 `12px`

## 5. 组件外观规范
- 按钮
  - 主按钮：实心品牌色
  - 次按钮：浅底+描边
  - 危险按钮：红色文字+浅红背景
- 表格
  - 表头背景略深于卡片背景
  - 行 hover 高亮使用低饱和蓝
- 表单
  - 标签固定中文文案，错误提示就近显示
  - 必填项显式标记

## 6. 页面级风格建议
- 知识库管理页：清晰表格+右侧配置抽屉
- 文档页：上传区做“投递面板”视觉，突出任务进度
- 问答页：双栏布局（会话侧栏 + 对话主区）
- 问答页由页面内侧栏承载会话导航，进入 `/app/ask` 时隐藏全局左侧导航，避免双侧栏重复。
- 问答页会话侧栏顶部保留品牌区（`CampusSage/用户端`），产品介绍文案通过品牌区 hover 浮层展示。
- 会话列表标题固定为“你的聊天”，位于品牌区下方；登录入口固定在侧栏底部用户块，不在顶部栏重复展示。
- 当登录角色为 `admin` 时，必须提供“管理端/用户端”双向切换入口（管理端头部与用户端侧栏均可触达）。
- 问答主区仅保留用户可理解控件（知识库选择、消息输入、发送/停止），不展示内部 ID 与高级调参项。
- 证据与耗时信息通过弹窗承载，点击助手消息后打开，不占用常驻右栏。
- 问答输入区采用底部浮层与吸底发送栏，消息滚动与输入操作保持解耦。
- 监控页：统计卡片+趋势图，强调异常色

## 7. 动效规范
- 全局动效时长：`160ms/240ms`
- 卡片入场：轻微上移 + 淡入
- 列表加载：骨架屏优于全局转圈
- 禁止过多微动效干扰阅读

## 8. 响应式规则
- `>=1200px`：双栏主布局
- `768px~1199px`：可切单栏，侧栏折叠
- `<768px`：优先保证问答与引用可读性，监控图表可降级

## 9. 可访问性
- 文本与背景对比满足可读要求。
- 所有可操作元素具备清晰 hover/focus 状态。
- 图标按钮必须有文本提示或 `aria-label`。

## 管理端去 ID 化约束（2026-02）
- 管理端页面（知识库、用户、文档、评测、监控）不展示内部主键标识（如 `kb_id/user_id/doc_id/job_id/run_id/eval_set_id`）。
- 管理端页面不提供“手动输入 ID”交互，用户只通过可读业务字段（名称、邮箱、时间、状态）完成筛选与操作。
- 各类 ID 仅允许作为前端内部值（`rowKey`、请求参数、状态缓存）使用，不进入默认可见文案。

## 管理端工作台补充（2026-03 第二轮）
- 文档页采用“文档投递 + 文档与任务”双栏工作台，上传、当前任务、历史任务在同一视图内完成。
- 文档与评测页的刷新、重建、重试、查询等高频动作优先使用“图标 + 短文案”按钮，减少整屏纯文字操作。
- 当前任务、评测结果等信息优先使用指标卡与状态标签表达，不直接暴露内部 ID。
- 评测页结果查询仅允许基于最近运行记录选择，不提供手动输入 `run_id/eval_set_id` 的默认交互。
- 会话运营页使用知识库名称下拉筛选与摘要式会话卡片，不在常态界面展示 `kb_id/conversation_id/request_id`。
- 管理端与用户端共用统一门户壳层：侧栏聚焦导航与身份信息，头部聚焦当前页面说明与端口切换。
- 布局层文案必须稳定使用中文，不允许在通用壳层中残留乱码式占位或英文调试文案。
- 知识库页与用户页应优先复用共享工作台组件，保持“左表单 / 右列表”结构与面板层级一致。
- 特殊布局页通过路由元信息声明，不允许在布局层新增基于 `pathname.startsWith(...)` 的页面特判。

## 共享工作台补充（2026-03 第三轮）
- 文档页与评测页继续复用 `OpsWorkbench + OpsPane`，保持管理端统一的“双栏工作台”结构。
- 面板标题下方如需承载筛选、密度切换、统计提示等轻量工具条，统一通过 `OpsPane.toolbar` 注入，不在页面内重复拼装相同容器。
- 上传、运行、列表、结果等主体内容统一放入滚动区，避免管理端不同页面出现不一致的面板层级。
