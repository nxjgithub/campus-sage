# UI_RULES.md — 前端视觉与样式规范

## 1. 视觉方向
整体视觉采用“学院数据中枢”风格：清晰、可信、理性，不使用花哨拟物，不走默认紫色系。

## 2. 品牌与色彩
建议 CSS 变量：
```css
:root {
  --bg-base: #f4f7fb;
  --bg-panel: #ffffff;
  --bg-emphasis: #e8f1ff;
  --text-primary: #10243e;
  --text-secondary: #4b5f78;
  --text-muted: #7b8ca4;
  --line-soft: #d9e3f0;
  --brand-main: #0f5bd8;
  --brand-accent: #0ea5a0;
  --ok: #1f9d55;
  --warn: #e5a100;
  --danger: #cf3f3f;
}
```

配色约束：
- 主操作使用 `--brand-main`。
- 状态色仅用于状态表达，不用于大面积装饰。
- 背景用浅灰蓝渐层，不用纯白大平面。

## 3. 字体规范
- 中文主字体：`"Source Han Sans SC", "Noto Sans SC", "PingFang SC", sans-serif`
- 英文与数字辅助字体：`"IBM Plex Sans", "Segoe UI", sans-serif`
- 代码与 ID：`"JetBrains Mono", "Cascadia Mono", monospace`

说明：
- 禁止默认 `Inter/Roboto/Arial` 作为唯一字体方案。
- 标题与正文需明确字号层级，不做全站同字号。

## 4. 排版与间距
- 8pt 栅格：`4/8/12/16/24/32`
- 页面最大宽度建议 `1200~1360px`
- 主内容区建议左右留白 `24~32px`
- 卡片圆角建议 `12px`

## 5. 组件外观规范
- 按钮
  - 主按钮：实心品牌色
  - 次按钮：浅底+描边
  - 危险按钮：红色文字+浅红背景
- 表格
  - 表头背景略深于卡片背景
  - 行 hover 高亮使用低饱和蓝
- 表单
  - 标签固定中文文案，错误提示就近显示
  - 必填项显式标记

## 6. 页面级风格建议
- 知识库管理页：清晰表格+右侧配置抽屉
- 文档页：上传区做“投递面板”视觉，突出任务进度
- 问答页：双栏布局
  - 左：对话流
  - 右：引用证据与耗时面板
- 监控页：统计卡片+趋势图，强调异常色

## 7. 动效规范
- 全局动效时长：`160ms/240ms`
- 卡片入场：轻微上移 + 淡入
- 列表加载：骨架屏优于全局转圈
- 禁止过多微动效干扰阅读

## 8. 响应式规则
- `>=1200px`：双栏主布局
- `768px~1199px`：可切单栏，侧栏折叠
- `<768px`：优先保证问答与引用可读性，监控图表可降级

## 9. 可访问性
- 文本与背景对比满足可读要求。
- 所有可操作元素具备清晰 hover/focus 状态。
- 图标按钮必须有文本提示或 `aria-label`。
