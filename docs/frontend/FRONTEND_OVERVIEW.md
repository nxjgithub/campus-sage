# FRONTEND_OVERVIEW.md — CampusSage 前端总览

## 1. 文档目的
本文件定义 CampusSage 前端的产品范围、页面边界与交付标准，用于约束后续人工开发与 AI 代码生成行为，避免功能漂移与重复返工。

## 2. 前端目标
- 构建可演示的高校知识库问答前端，完整覆盖“建库→上传→入库→问答→引用→反馈→会话回放”闭环。
- 对接现有后端接口（`/api/v1`），不引入与后端契约冲突的私有字段。
- 保障可追溯性：所有失败提示可显示 `error.code` 与 `request_id`。
- 保障可解释性：问答结果必须展示引用证据或明确拒答原因。

## 3. 非目标
- 不做复杂审批流或组织架构管理（本期仅覆盖用户与 KB 访问控制）。
- 不做重型内容管理后台（仅覆盖答辩所需核心能力）。
- 不做移动端原生 App（仅响应式 Web）。

## 4. 用户与角色
- 管理员：创建知识库、上传文档、查看入库任务、查看队列监控。
- 管理员：创建知识库、上传文档、查看入库任务、查看队列监控、管理用户与 KB 访问权限。
- 普通用户：在指定知识库提问、查看引用、提交反馈、查看会话历史。

路由分端约定：
- 管理员端：`/admin/*`
- 用户端：`/app/*`
- 登录页：`/login`

## 5. 页面清单（必须实现）
- `管理员端（/admin）`
  - 用户管理页
  - 创建用户
  - 更新用户状态/角色/密码
  - 查看并设置用户知识库访问权限（read/write/admin）
  - 撤销用户单条知识库权限
  - 批量覆盖用户知识库权限
  - 知识库管理页
  - 创建知识库
  - 查看知识库列表
  - 编辑知识库配置（`topk/threshold/rerank_enabled/max_context_tokens`）
  - 删除知识库
  - 文档管理页
  - 上传文档（仅 `pdf`）
  - 查看文档列表
  - 删除文档
  - 发起重建索引
  - 入库任务页/侧栏
  - 查看任务状态与进度
  - 取消任务
  - 重试任务
  - 队列监控页
  - 展示队列统计
  - 执行失败任务转死信队列操作
  - 评测中心
  - 创建评测集
  - 发起评测运行
  - 查询评测结果
- `用户端（/app）`
  - 问答页
  - 基于指定知识库提问
  - 支持匿名问答 public 知识库（可手动输入 `kb_id`）
  - 展示答案、拒答状态、建议、引用证据
  - 可选传入 `topk/threshold/rerank_enabled/filters/debug`
  - 会话历史页
  - 会话列表查询（可按 `kb_id` 过滤）
  - 查看会话消息与引用回放
  - 反馈入口
  - 针对问答消息提交 `up/down` 反馈
  - 记录 `reasons/comment/expected_hint`
- `认证页（/login）`
  - 邮箱密码登录
  - 登录成功后按角色跳转（admin→`/admin/kb`，user→`/app/ask`）
  - 支持携带 `next` 参数回跳原页面

## 6. 前后端契约原则
- 只以后端开放接口为准，不在前端伪造状态。
- 认证采用 JWT（access + refresh），由前端自动刷新 access_token。
- `refusal=true` 属于业务正常结果，不按接口失败处理。
- 所有接口异常统一使用后端错误结构：
  - `error.code`
  - `error.message`
  - `error.detail`
  - `request_id`

## 7. 交付里程碑
- M1：知识库 + 文档上传 + 入库任务轮询可用。
- M2：问答页（含引用、拒答、反馈）可用。
- M3：会话历史 + 队列监控 + 全链路错误处理可用。
- M4：视觉统一、响应式适配、演示脚本就绪。

## 8. 前端完成标准（DoD）
- 每个页面至少有 `loading/success/empty/error` 四态处理。
- 受保护页面必须具备登录守卫与未登录跳转。
- 问答页必须正确区分“正常回答”和“拒答”。
- 引用卡片必须展示 `doc_name + (page 或 section_path) + snippet`。
- 接口报错必须可见 `request_id`。
- 文档、接口、代码实现一致。
