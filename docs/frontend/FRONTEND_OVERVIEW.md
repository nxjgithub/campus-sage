# FRONTEND_OVERVIEW.md

## 1. 目标
CampusSage 前端用于完成以下闭环：
- 知识库治理
- 文档上传与入库
- 证据问答
- 引用查看
- 反馈回收
- 会话回放
- 监控与评测

## 2. 角色与入口
- 管理员端：`/admin/*`
- 用户端：`/app/*`
- 登录页：`/login`

管理员负责治理、入库、评测、监控、用户与权限。
用户负责提问、查看引用、提交反馈、管理会话。

## 3. 页面清单

## 4. 前后端契约原则
- 以前后端文档与 OpenAPI 为准，不伪造字段。
- `refusal=true` 是正常业务结果，不按失败处理。
- 所有错误提示必须可关联 `request_id`。
- 所有引用展示必须包含：
  - `doc_name`
  - `page` 或 `section_path`
  - `snippet`

## 5. 当前前端能力
### 管理端
- 知识库列表维护、独立创建、编辑、删除、策略配置
- 用户列表维护、独立创建、状态编辑、角色配置、知识库授权
- 文档列表维护、独立上传、任务查看、取消、重试、重建索引
- 评测运行、评测结果查看、独立评测集创建
- 队列监控、失败任务迁移

### 用户端
- 选择知识库提问
- 流式回答
- 引用弹窗
- 拒答建议
- 会话列表、重命名、删除
- 反馈、重试、改写后重发
- 问答侧栏上下文概览、会话状态卡片与底部快捷问题填充

## 6. 近期界面方向
- 门户壳层统一为左侧侧栏工作台。
- 管理端页面统一为紧凑页头 + 双栏工作台 + 图表摘要。
- 复合型管理页优先拆成“管理页 + 独立创建或上传页”，避免录入表单和结果表格互相挤压。
- 用户端问答页统一为聊天式布局，摘要图只做轻量补充。
- 用户端问答页继续强化“侧栏上下文 + 主消息轨道 + 底部提问区”三段结构，首屏优先传达知识库范围、证据约束与快捷提问入口。

## 7. 完成标准
- 每个页面必须具备 `loading / success / empty / error` 四态。
- 高风险操作必须二次确认。
- 文档、代码、行为必须同步。
- 图表只增强判断，不替代表格和精确数据。
